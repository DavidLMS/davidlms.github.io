Práctica: Servidor DNS en Ubuntu Server 16.04

El Servicio de Nombres de Dominio (Domain Name Service, DNS) es un servicio de Internet que hace corresponder direcciones IP con nombres de dominio totalmente cualificados (FQDN), unos con otros. De esta forma, DNS evita tener que recordar direcciones IP. Los equipos que ejecutan DNS se denominan servidores de nombres. Ubuntu trae el BIND (Berkley Internet Naming Daemon), el programa más usado para mantener un servidor de nombres en Linux.

Clasificación de servidores DNS

Podemos clasificarlos de dos formas:

  • Servidor primario y secundario.
  • Servidor maestro, esclavo y caché.

Servidor primario y secundario

El servidor primario será el servidor DNS por defecto al que se realizará la consulta. El servidor secundario es otro servidor DNS que actuará como tal cuando el primario no esté disponible.

Servidor maestro, esclavo y caché

El servidor maestro es un servidor DNS que funciona de forma autónoma. El servidor esclavo solamente es un clon del anterior. Esto quiere decir que cualquier cambio en la configuración del maestro se verá reflejado automáticamente en el esclavo. Esta combinación se utiliza para añadir redundancia y eficiencia.
El servidor caché es aquel intermediario entre los clientes y el servidor DNS. Cuando un cliente hace una consulta, el servidor caché registra esa consulta al servidor DNS, de forma que si otro cliente realiza la misma consulta, le responde el servidor caché y no es necesario volver a preguntar al servidor DNS. Se suele utilizar en redes locales para agilizar el acceso a los diferentes dominios.

Instalación

En un terminal, introduzca el siguiente comando para instalar dns:

sudo apt-get install bind9

Un paquete muy útil para testear y encontrar problemas relacionados con DNS es dnsutils. Normalmente se instalará con el anterior, pero lo podemos comprobar ejecutando:

sudo apt-get install dnsutils

Configuración DNS como maestro primario

En esta práctica, BIND9 será configurado como el Maestro Primario (Primary Master) para el dominio mundonuevo.com. Simplemente sustituye mundonuevo.com por su nombre de dominio totalmente cualificado (Fully Qualified Domain Name, FQDN).

Archivo de zona de redirección

Para agregar una zona DNS a BIND0, cambiando a BIND9 en un servidor Maestro Primario, el primer paso es editar /etc/bind/named.conf.local:

zone "mundonuevo.com" {
    type master;
    file "/etc/bind/db.mundonuevo.com";
};

Ahora utiliza un archivo de zona existente como una plantilla para crear el archivo /etc/bind/db.mundonuevo.com:

sudo cp /etc/bind/db.local /etc/bind/db.mundonuevo.com

Edita el nuevo archivo de zona /etc/bind/db.mundonuevo.com cambiando todas las apariciones de localhost por mundonuevo.com dejando el “.” adicional al final. Cambia 127.0.0.1 por la dirección IP de nuestro servidor y root.localhost por un correo electrónico válido, pero utilizando un “.” en lugar de la “@”.

Crea un registro A para el dominio base, mundonuevo.com. Crea también un registro A para ns.mundonuevo.com, el servidor de nombres en este ejemplo:

;
; BIND data file for mundonuevo.com
;
$TTL    604800
@       IN      SOA     mundonuevo.com. root.mundonuevo.com. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
        IN      A       192.168.1.10
;
@       IN      NS      ns.mundonuevo.com.
@       IN      A       192.168.1.10
@       IN      AAAA    ::1
ns      IN      A       192.168.1.10

Debes incrementar el Serial cada vez que hagas cambios en el archivo de zona. Si haces múltiples cambios antes de reiniciar BIND9, simplemente increméntalo una vez.

Una vez realizados los cambios, será necesario reiniciar BIND9:

sudo systemctl restart bind9.service

Archivo de Zona Reversa

Ahora que se ha configurado la zona y se resuelven nombres a direcciones IP, también se necesita una zona inversa (Reverse zone). Una zona inversa permite al DNS resolver una dirección a un nombre.

Edita /etc/bind/named.conf.local y añade lo siguiente:

zone "1.168.192.in-addr.arpa" {
        type master;
        file "/etc/bind/db.192";
};

Reemplazando “1.168.192” por los tres primeros octetos de la IP del servidor en orden inverso.

Ahora crea el archivo /etc/bind/db.192:

sudo cp /etc/bind/db.127 /etc/bind/db.192

Luego edita /etc/bind/db.192 cambiando básicamente las mismas opciones que en /etc/bind/db.mundonuevo.com:

;
; BIND reverse data file for local 192.168.1.XXX net
;
$TTL    604800
@       IN      SOA     ns.mundonuevo.com. root.mundonuevo.com. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;
@       IN      NS      ns.
14      IN      PTR     ns.mundonuevo.com.

Reemplazando el “14” por el último octeto de la IP del servidor.

Volvemos a reiniciar BIND9:

sudo systemctl restart bind9.service

Ya está listo nuestro servidor de nombres. En el siguiente apartado probaremos que funciona correctamente a través de una página web alojada en nuestro servidor.

Prueba del servidor de nombres

Podemos probar el correcto funcionamiento del servidor DNS poniéndolo como servidor en algún ordenador cliente y utilizando el comando nslookup, que funciona tanto en Windows como en UNIX para obtener la dirección IP conociendo el nombre, y viceversa.

nslookup mundonuevo.com

Podemos comprobar que nos devuelve la IP asignada en nuestro servidor DNS. También podemos utilizar el comando dig, que proporciona más información.

dig mundonuevo.com

Cambiar servidor DNS en Linux

Editamos el archivo /etc/resolv.conf y ponemos la línea:
nameserver 192.168.1.14
Siendo 192.168.1.14 la dirección IP de nuestro servidor de nombres.

Debemos recordar dejar el archivo como estaba previamente después de probar el DNS.

Cambiar servidor DNS en Windows

Hacemos click derecho en el icono de la red y pulsamos en Abrir el Centro de redes y recursos compartidos.

A continuación le damos a Cambiar configuración del adaptador.

Hacemos click derecho en la interfaz de red que estemos utilizando y pulsamos en Propiedades.

Seleccionamos Protocolo de Internet versión 4 (TCP/IPv4) y le damos a Propiedades.

Seleccionamos Usar las siguientes direcciones de servidor DNS: y escribimos en el campo Servidor DNS preferido la IP de nuestro servidor de nombres.

Aceptamos todo. Para asegurarnos de que esta configuración tenga efecto, en la ventana en la que aparecen todas las interfaces de redes seleccionamos la que estamos utilizando y pulsamos en Deshabilitar este dispositivo de red y luego volvemos a habilitarlo.

Debemos recordar dejar la configuración como estaba previamente después de probar el DNS.

Cambiar servidor DNS en macos

Pulsamos en el icono de la red en la esquina superior derecha.

Le damos a Abrir el panel de preferencias Red.

Pulsamos en Avanzado.

En la pestaña DNS, le damos al icono + en el apartado Servidores DNS y escribimos la dirección IP de nuestro servidor DNS.

Pulsamos en Aceptar y Aplicar.

Para asegurarnos que estamos utilizando la configuración que acabamos de poner, podemos Desactivar y Activar Wi-Fi en el botón arriba a la derecha.

Debemos recordar dejar la configuración como estaba previamente después de probar el DNS.
